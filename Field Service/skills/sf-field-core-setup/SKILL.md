# sf-field-core-setup

Field Service baseline setup strategy and data-model alignment.

Use when users need to define or review foundational Field Service configuration: service territories, operating hours, service resources, skills, work types, and scheduling policies.

TRIGGER when:
- User asks to stand up Field Service from scratch.
- User needs territory hierarchy or operating-hour design.
- User is mapping business process to Field Service core objects.

DO NOT TRIGGER when:
- Task is focused only on mobile UX (use `sf-field-mobile-technician`).
- Task is focused only on optimization tuning (use `sf-field-optimization-capacity`).
- Task is focused only on communications (use `sf-field-customer-communications`).

Primary outcomes:
- Baseline object and relationship blueprint.
- Environment readiness checklist.
- Dependency-aware rollout plan.
