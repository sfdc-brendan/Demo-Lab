# sf-field-review

Review rubric for Field Service architecture, operations, and adoption readiness.

Use when users ask for a review, audit, or quality check of a Field Service solution before go-live or scale-out.

TRIGGER when:
- User asks to review setup for risk or production readiness.
- User asks for architecture and operational gap analysis.
- User asks for prioritized remediation plan.

Review dimensions:
- Data model integrity and lifecycle correctness.
- Scheduling/dispatch reliability and exception handling.
- Mobile execution quality and offline resilience.
- Inventory and SLA alignment with operational outcomes.
- Governance, KPIs, and adoption risk.

Primary outcomes:
- Severity-ranked findings.
- Concrete fixes and rollout sequencing.
- Residual-risk statement and test recommendations.
