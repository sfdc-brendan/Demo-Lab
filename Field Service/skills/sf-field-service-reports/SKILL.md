# sf-field-service-reports

Service report and proof-of-service operating model for compliance and customer trust.

Use when users need report templates, required capture points, signature policy, and closure package standards.

TRIGGER when:
- User asks what technicians must document before close.
- User asks for signature/photo/report requirements.
- User asks for post-job record quality and auditability.

DO NOT TRIGGER when:
- Task is customer notification orchestration (use `sf-field-customer-communications`).
- Task is work-order lifecycle design only (use `sf-field-work-order-lifecycle`).

Primary outcomes:
- Required evidence model by work type.
- Report generation and distribution standards.
- Compliance checks before job completion.
