# Customer Success Frameworks

Operational frameworks for health scoring, adoption maturity, expansion strategy, QBR execution, risk management, renewal preparation, and Voice of the Customer synthesis.

---

## Health Score Model

A composite health score across six weighted dimensions. Each dimension scores 0–100, weighted to produce a composite score.

### Dimensions and Weights

| Dimension | Weight | What It Measures | Data Sources |
|---|---|---|---|
| **Adoption** | 25% | Feature usage, login frequency, workflow completion, license utilization | Login reports, feature usage analytics, license assignment data |
| **Support Health** | 20% | Ticket volume trend, resolution time, P1/P2 frequency, escalation rate | Case data, support dashboards |
| **Executive Engagement** | 15% | Sponsor meeting attendance, champion activity level, executive responsiveness | CSM notes, meeting cadence tracker |
| **Satisfaction (NPS/CSAT)** | 15% | Survey scores, qualitative feedback sentiment, reference willingness | NPS surveys, CSAT data, feedback log |
| **Contract Utilization** | 15% | Licenses used vs. entitled, features purchased vs. deployed, storage usage | License reports, entitlement data |
| **Growth Trajectory** | 10% | User growth rate, new use case adoption, expansion conversations, strategic alignment | User count trends, expansion pipeline, roadmap alignment |

### Scoring Thresholds per Dimension

| Score Range | Status | Definition |
|---|---|---|
| 80–100 | 🟢 Healthy | Strong performance, no action needed, proactively optimizing |
| 60–79 | 🟡 Attention | Below target, specific areas need intervention, not yet critical |
| 0–59 | 🔴 At Risk | Significant concern, immediate action required, risk to renewal |

### Composite Score Interpretation

| Composite Score | Overall Health | CSM Action |
|---|---|---|
| 85–100 | Thriving | Maintain cadence, focus on expansion and advocacy |
| 70–84 | Healthy | Address attention dimensions, prepare for expansion |
| 55–69 | Needs Attention | Focused intervention plan, increase cadence, alert AE |
| 40–54 | At Risk | Risk mitigation plan, executive escalation, weekly monitoring |
| 0–39 | Critical | All-hands intervention, VP engagement, save plan |

### Scoring Guidelines by Dimension

#### Adoption (25%)
| Score | Criteria |
|---|---|
| 90–100 | 85%+ WAU, all key features adopted, self-service reporting active |
| 70–89 | 65–84% WAU, most features adopted, some underused areas |
| 50–69 | 45–64% WAU, core features used but advanced features ignored |
| 30–49 | 25–44% WAU, only basic features used, declining trend |
| 0–29 | Below 25% WAU, minimal usage, shelfware risk |

#### Support Health (20%)
| Score | Criteria |
|---|---|
| 90–100 | Low ticket volume, fast resolution, zero P1s this quarter, declining trend |
| 70–89 | Moderate volume, acceptable resolution time, rare P1, stable |
| 50–69 | Elevated volume, some SLA misses, occasional P1, flat trend |
| 30–49 | High volume, frequent SLA misses, multiple P1s, rising trend |
| 0–29 | Critical volume, persistent SLA failures, unresolved P1s, customer frustration visible |

#### Executive Engagement (15%)
| Score | Criteria |
|---|---|
| 90–100 | Sponsor actively participates, proactively advocates, multiple champions |
| 70–89 | Sponsor attends scheduled meetings, responsive, one strong champion |
| 50–69 | Sponsor delegates to junior staff, slow response, champion exists but passive |
| 30–49 | Sponsor cancelling meetings, unresponsive, no clear champion |
| 0–29 | No executive engagement, contact lost, no internal advocacy |

#### Satisfaction — NPS/CSAT (15%)
| Score | Criteria |
|---|---|
| 90–100 | NPS 50+, CSAT 4.5+/5, willing reference, positive qualitative feedback |
| 70–89 | NPS 30–49, CSAT 4.0–4.4, generally positive, minor concerns |
| 50–69 | NPS 10–29, CSAT 3.5–3.9, mixed sentiment, specific complaints |
| 30–49 | NPS -10–9, CSAT 3.0–3.4, multiple complaints, dissatisfaction visible |
| 0–29 | NPS below -10, CSAT below 3.0, actively unhappy, detractor risk |

#### Contract Utilization (15%)
| Score | Criteria |
|---|---|
| 90–100 | 90%+ license utilization, all purchased features deployed, storage healthy |
| 70–89 | 75–89% utilization, most features deployed, minor waste |
| 50–69 | 50–74% utilization, some purchased features undeployed, shelfware risk |
| 30–49 | 30–49% utilization, significant shelfware, questions about value |
| 0–29 | Below 30% utilization, major shelfware, renewal justification difficult |

#### Growth Trajectory (10%)
| Score | Criteria |
|---|---|
| 90–100 | Active expansion pipeline, new use cases deploying, strategic alignment strong |
| 70–89 | Expansion discussed, some new use cases planned, generally aligned |
| 50–69 | No active expansion, maintaining current scope, neutral alignment |
| 30–49 | Contracting signals, reducing scope, questioning value |
| 0–29 | Active reduction conversations, RFP for alternatives, misalignment |

## Adoption Maturity Model

Five stages of customer adoption maturity. Each stage has clear criteria, CSM focus areas, and graduation triggers.

### Stage 1: Onboarding (Months 0–3)

**Characteristics:**
- Just went live or recently purchased new cloud
- Users learning basic navigation and core workflows
- Data migration in progress or recently completed
- High support ticket volume (expected)
- Executive sponsor engaged from sale momentum

**CSM Focus:**
- Ensure smooth technical handoff from implementation team
- Validate data quality and completeness
- Track first-login metrics across all user segments
- Establish communication cadence with champion and sponsor
- Set baseline metrics for success plan

**Graduation Criteria:**
- [ ] 90%+ first login achieved
- [ ] Core workflows documented and trained
- [ ] Zero critical (P1) open issues
- [ ] Executive sponsor confirmed for ongoing cadence
- [ ] Baseline adoption scorecard produced

### Stage 2: Basic Adoption (Months 3–6)

**Characteristics:**
- Users performing core daily tasks in Salesforce
- Basic reporting in use by managers
- Support tickets shifting from "how do I" to "can I also"
- Some power users emerging naturally
- Data entry habits forming (good or bad)

**CSM Focus:**
- Drive weekly active user rate above 65%
- Identify and nurture emerging champions
- Address data quality issues before they compound
- Deliver first QBR with baseline-to-current comparison
- Introduce next-tier features appropriate to maturity

**Graduation Criteria:**
- [ ] 65%+ weekly active user rate
- [ ] At least 2 identified champions per business unit
- [ ] Manager-level reporting adopted (dashboards viewed weekly)
- [ ] Data quality metrics within threshold
- [ ] First QBR delivered with positive reception

### Stage 3: Proficient Use (Months 6–12)

**Characteristics:**
- Salesforce is the system of record — users default to it
- Automations running reliably, reducing manual work
- Self-service reporting adopted by business leaders
- Feature requests indicate users want more capability
- Integration with other systems operational

**CSM Focus:**
- Push toward 80%+ adoption across core features
- Capture first ROI data points and value stories
- Begin expansion opportunity assessment
- Introduce advanced features (AI, analytics, automation)
- Develop internal advocacy (case studies, references)

**Graduation Criteria:**
- [ ] 80%+ weekly active user rate
- [ ] At least one quantified ROI metric
- [ ] Advanced feature pilot launched or planned
- [ ] Customer willing to be a peer reference
- [ ] Expansion opportunity map drafted

### Stage 4: Advanced Use (Months 12–24)

**Characteristics:**
- Advanced features in active use (Einstein, complex automations, advanced reporting)
- Platform integral to multiple business processes
- Customer driving their own roadmap, not just following CSM guidance
- Internal Salesforce CoE or admin team maturing
- Clear business value demonstrated and shared at executive level

**CSM Focus:**
- Optimize and fine-tune for efficiency gains
- Support customer-driven innovation and roadmap
- Drive expansion opportunities through demonstrated value
- Prepare multi-year success plan aligned to customer's strategic plan
- Strengthen executive relationship beyond the immediate sponsor

**Graduation Criteria:**
- [ ] 85%+ adoption, multiple advanced features active
- [ ] Customer presents their own Salesforce success at internal meetings
- [ ] Multi-cloud expansion in progress or completed
- [ ] Internal CoE self-sufficient for day-to-day operations
- [ ] ROI exceeds original business case

### Stage 5: Strategic Platform (24+ Months)

**Characteristics:**
- Salesforce is a strategic investment, not just a tool
- C-suite visibility — Salesforce outcomes in board discussions
- Multi-cloud, deeply integrated, business-critical
- Customer co-innovating with Salesforce (beta programs, advisory boards)
- Platform decisions tied to enterprise strategy, not just departmental needs

**CSM Focus:**
- Strategic advisor mode — quarterly executive alignment
- Innovation partnership (connecting customer to PM, beta programs)
- Enterprise-wide expansion and optimization
- Advocacy programs (references, events, case studies)
- Succession planning for key stakeholders to prevent relationship risk

**Graduation Criteria:**
This is the target state. The CSM's goal is to maintain and deepen strategic positioning, preventing regression to lower maturity stages.

## Expansion Playbook Patterns

Four proven expansion motions, each suited to different customer contexts.

### Pattern 1: Land-and-Expand by Cloud

**When to use:** Customer started with one cloud, has achieved proficient use (Stage 3+), and has adjacent business processes that map to another Salesforce cloud.

**Approach:**
1. Identify the next-best-fit cloud from whitespace analysis
2. Connect current success metrics to the expansion business case ("Sales Cloud reduced X, imagine the same impact on service")
3. Arrange DSE-led architecture session to show cross-cloud value
4. Propose pilot or phased rollout, not big-bang
5. Leverage existing champions to introduce new stakeholders

**Key Principle:** The best expansion pitch is a success story from the cloud they already own.

### Pattern 2: User-Led Growth

**When to use:** Customer's current deployment succeeds, and new teams, regions, or business units want the same capability.

**Approach:**
1. Monitor for hiring triggers, org restructuring, or new office openings
2. Quantify per-user value from existing deployment
3. Propose same-cloud expansion with configuration for new segment
4. Frame as extending proven value, not a new project
5. Request champion from existing deployment to advocate to new segment

**Key Principle:** Internal word-of-mouth is the most credible sales motion. Arm the champion with data.

### Pattern 3: Executive-Sponsored Expansion

**When to use:** Executive sponsor has a strategic initiative that maps to a Salesforce capability not yet deployed.

**Approach:**
1. Listen for strategic themes in QBRs and executive conversations
2. Connect the strategic initiative to a Salesforce capability with a clear value hypothesis
3. Prepare a one-page executive brief (aligned to sf-dse VoC brief format)
4. Arrange a strategic planning session with DSE involvement
5. Position as Salesforce enabling the strategy, not Salesforce selling a product

**Key Principle:** The expansion must serve the executive's agenda, not Salesforce's.

### Pattern 4: Event-Triggered Expansion

**When to use:** An external or internal event creates an immediate need that a Salesforce product addresses.

**Common triggers:**
- M&A activity (need to consolidate systems)
- Regulatory change (need compliance capabilities)
- Competitive pressure (need customer experience improvement)
- Leadership change (new executive with different priorities)
- Technology sunset (legacy system end-of-life)

**Approach:**
1. Maintain a trigger event watchlist for each customer
2. When trigger fires, move quickly — be the first call, not the last
3. Connect the event to a specific Salesforce capability with urgency
4. Frame as "we can help you respond to this" not "you should buy this"
5. Bring AE and DSE into the conversation early

**Key Principle:** Speed matters. The value of event-triggered expansion decays rapidly.

## QBR Best Practices

### Preparation Checklist (2 Weeks Before)

- [ ] Pull adoption data for the period and compare to prior period
- [ ] Update health score with current dimension-level data
- [ ] Review success plan — which milestones were hit or missed?
- [ ] Check support ticket log for themes, P1/P2 resolution status
- [ ] Review any expansion opportunities or risks to discuss
- [ ] Identify one "insight spotlight" — data the customer doesn't know yet
- [ ] Prepare next-quarter goals and mutual commitments draft
- [ ] Pre-align with AE on commercial topics (if any)
- [ ] Send agenda to customer at least 1 week before the meeting
- [ ] Confirm attendee list — push for executive sponsor attendance

### Agenda Structure (60-Minute QBR)

| Time | Section | Purpose |
|---|---|---|
| 0–5 min | Welcome & agenda | Set the tone, confirm objectives for the meeting |
| 5–15 min | Period in review | Celebrate wins, show metrics, acknowledge challenges |
| 15–25 min | Adoption deep-dive | Feature-level data, trends, segments |
| 25–35 min | Insight spotlight | One "aha" the customer didn't know — this is the differentiator |
| 35–45 min | Roadmap alignment | What's coming from Salesforce that matters to this customer |
| 45–55 min | Next-quarter goals | Mutual commitments with clear owners and dates |
| 55–60 min | Open discussion | Customer raises topics, CSM captures action items |

### Presenting Data That Drives Action

**Do:**
- Show trends over time, not just snapshots
- Compare to targets, not just prior period
- Segment data meaningfully (by team, role, region)
- Pair every metric with a "so what" interpretation
- Include one benchmark (industry or Salesforce peer group) if available
- Lead with wins before addressing areas needing attention

**Don't:**
- Dump 20 metrics without interpretation
- Use Salesforce jargon the business audience doesn't understand
- Present problems without proposed solutions
- Make the customer feel graded — QBR is a partnership review, not a report card
- Skip the "insight spotlight" — this is what separates a great QBR from a status meeting

### Avoiding "Report Card" Syndrome

The single biggest QBR anti-pattern is turning the meeting into a pass/fail review where the customer feels judged. To avoid it:

1. **Frame as "our joint progress"** not "your performance"
2. **Lead with value delivered** before discussing gaps
3. **When presenting low metrics, pair with a plan** — "here's what we'll do together"
4. **Ask for the customer's perspective** before presenting your conclusions
5. **End with mutual commitments** — both sides have action items, not just the customer

## Risk Signal Taxonomy

### Red Signals (Immediate Action Required)

| Signal | Threshold | Why It Matters |
|---|---|---|
| Executive sponsor lost or disengaged | 3+ cancelled meetings, no response to outreach | Budget authority and strategic alignment at risk |
| WAU drops below 40% | Sustained for 2+ weeks | Shelfware trajectory, renewal justification weakens |
| P1 support case unresolved > 5 business days | Any P1 open beyond SLA | Erosion of trust, potential executive escalation |
| Champion departure | Primary champion leaves the company | Institutional knowledge and internal advocacy lost |
| Customer evaluating competitors | RFP activity or competitive vendor meetings | Active churn risk |
| Contract reduction request | Customer formally asks to reduce scope | Commercial risk, value not demonstrated |

### Yellow Signals (Elevated Monitoring)

| Signal | Threshold | Why It Matters |
|---|---|---|
| WAU declining 3+ consecutive periods | Trend direction matters more than absolute number | Leading indicator of deeper disengagement |
| Support ticket volume rising | 20%+ increase quarter-over-quarter | Could indicate product issues, training gaps, or frustration |
| Executive sponsor delegates to junior staff | Pattern of delegation, not one-off | Decreasing priority, may indicate dissatisfaction |
| Key stakeholder turnover | Any stakeholder in RACI changes roles | Relationship risk, institutional knowledge loss |
| NPS/CSAT below 30 / 3.5 | Survey data below threshold | Sentiment turning, renewal negotiation will be harder |
| Purchased features not deployed after 6 months | Shelfware indicator | Customer not getting value from what they bought |

### Green Indicators (Healthy — Maintain and Optimize)

| Signal | Threshold | Why It Matters |
|---|---|---|
| WAU stable or growing above 75% | Sustained | Strong adoption, users find value |
| NPS above 40 / CSAT above 4.2 | Sustained | Customer is a potential advocate |
| Executive sponsor proactively engages | Initiates meetings or requests | Strategic alignment, sponsor sees value |
| Feature requests increasing | Quality requests, not complaints | Customer wants more, not less — expansion signal |
| Customer willing to be a reference | Offers or agrees when asked | Ultimate trust indicator |
| License utilization above 85% | Sustained | Getting value from investment, may need more |

## Renewal Preparation Timeline

A 12-month countdown to a successful renewal, starting from 12 months before the contract anniversary.

### T-12 Months: Baseline

- [ ] Confirm contract terms, renewal date, and pricing model
- [ ] Review current health score and adoption data
- [ ] Document value delivered since last renewal
- [ ] Identify any unresolved risks or open issues
- [ ] Align with AE on renewal strategy and pricing approach

### T-9 Months: Value Evidence

- [ ] Compile quantified ROI data (time saved, revenue impact, cost reduction)
- [ ] Collect qualitative success stories from stakeholders
- [ ] Deliver QBR focused on value demonstrated and future roadmap
- [ ] Address any red or yellow health score dimensions
- [ ] Confirm executive sponsor engagement and satisfaction

### T-6 Months: Strategic Alignment

- [ ] Review customer's strategic plan for the next year
- [ ] Map Salesforce capabilities to customer's upcoming priorities
- [ ] Present a forward-looking success plan tied to renewal
- [ ] Surface any expansion opportunities that strengthen the renewal case
- [ ] Pre-empt competitor positioning by reinforcing switching costs and sunk investment

### T-4 Months: Commercial Preparation

- [ ] AE begins pricing and commercial discussion
- [ ] CSM provides value summary and adoption data to support negotiation
- [ ] Address any outstanding issues that could be used as negotiation leverage
- [ ] Prepare executive business review if not done recently
- [ ] Ensure all decision-makers are identified and engaged

### T-2 Months: Finalization

- [ ] Commercial terms agreed or in final negotiation
- [ ] CSM confirms no unresolved risks or blockers
- [ ] Customer has visibility into post-renewal plan (roadmap, goals)
- [ ] Internal approvals in progress on customer side
- [ ] Legal review (if required) initiated

### T-0: Renewal

- [ ] Contract signed
- [ ] Post-renewal success plan launched for next period
- [ ] Celebrate with the customer — acknowledge the partnership
- [ ] Internal debrief: what worked, what to improve for next renewal

## Voice of the Customer (VoC) Synthesis

How to aggregate customer feedback into actionable themes and feed them into the product and strategy loop.

### Collection Sources

| Source | Frequency | Capture Method |
|---|---|---|
| QBR conversations | Quarterly | Meeting notes → VoC log |
| Support tickets (themes) | Monthly | Ticket analysis → trend report |
| NPS/CSAT verbatims | Per survey | Survey responses → sentiment tagging |
| Champion conversations | Ongoing | CSM notes → VoC log |
| Feature requests | Ongoing | Request tracker → VoC log |
| Executive conversations | Monthly | CSM notes → VoC log |
| User community / Idea Exchange | Monthly | Community monitoring → VoC log |

### VoC Log Structure

| Date | Customer | Source | Feedback Theme | Verbatim Quote | Sentiment | Product Area | Action Taken |
|---|---|---|---|---|---|---|---|
| [Date] | [Customer] | [QBR] | [Reporting complexity] | ["We need simpler dashboards for frontline managers"] | [Negative] | [Analytics] | [Flagged to PM] |

### Synthesizing Themes

1. **Aggregate** — Collect all VoC entries for the review period
2. **Categorize** — Group by product area, sentiment, and request type (bug, enhancement, new feature)
3. **Count** — Identify the top themes by frequency across customers
4. **Weight** — Apply customer size, strategic importance, and renewal proximity as weighting factors
5. **Contextualize** — Add business impact for the top themes (revenue at risk, adoption blocked, etc.)
6. **Recommend** — For each top theme, propose an action (product fix, enablement, process change, or workaround)

### Connecting to DSE VoC Brief Format

When escalating aggregated VoC to the DSE for strategic product feedback:

| VoC Brief Field | CSM Provides |
|---|---|
| Theme | Top-3 themes by weighted frequency |
| Customer evidence | Count of customers affected, representative quotes |
| Business impact | Revenue at risk, adoption blocked, competitive exposure |
| Recommended action | Product fix, roadmap request, or interim workaround |
| Urgency | Timeline based on renewal proximity and churn risk |

This feeds directly into the DSE's product strategy conversations and account-level planning.
